Infosec theory and encryption (4 points)

1.  What are the three pillars of information security? Describe each in a sentence.

    Confidentiality: Secure information so that sensitive information is only accessible by those who are authorized.

    Integrity: Guarantee the information is accurate and unaltered during storage or transmission.
    
    Availability: Ensure information and resources are accessible to authorized users when needed.
 
    -   A hacker breaks into a hospital system and encrypts all patient records until receipt of payment. 
        This is known as a /ransomware/ attack. What pillar of information security is undermined by this attack?

        A ransomware attack undermines the availability pillar of information security because it prevents authorized users from accessing information that is needed by encrypting it for ransom.

    -   A hacker breaks into a payment processor's database, extracts all its information, and sells it to criminals.
        What pillar of information security is undermined by this attack?

        The pillar of confidentiality is undermined by this attack because the hacker is an unauthorized actor accessing sensitive information that was intended to be confidential.

2.  Explain to what end and under what circumstances two parties would use the Diffie-Hellman key exchange algorithm.

    Diffie-Hellman should be used when trying to established a shared secret key for encrypted commmunication between two parties without having to transmit the key itself using mathematical operations to end up with the same result for both users.

4.  Assume that you and I have a preshared key K. Explain how you can used this key to attach an authentication code to a message that certifis to me that you authored the message /without encrypting the message/.

    Using the preshared key K, I compute a message authentication code (MAC) on the message and send both the message and the MAC; the receiver recomputes the MAC using K, and a match verifies that the message was authored by someone who knows the key, without encrypting the message.
